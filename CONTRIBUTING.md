# Contributing to AIearth

Thank you for your interest in contributing to this project. Here are a few ways you can help.

## Reporting Issues

Open a GitHub issue and include:
- A clear title and description of the problem
- Steps to reproduce, if applicable
- Browser/environment details for visualization bugs

## Suggesting New Data Sources

We welcome suggestions for additional survey data or indices. Please open an issue with:
- The name and URL of the source
- Which countries it covers
- The year and methodology used
- Why it would improve coverage or accuracy

## Submitting Pull Requests

1. Fork the repository and create a feature branch from `main`.
2. Make your changes and ensure the data validates against the existing JSON schema.
3. Update `DATA_DICTIONARY.md` if you add or modify any fields.
4. Open a pull request with a clear description of what changed and why.

## Code Style

- Use 2-space indentation for JSON and JavaScript files.
- Keep data entries sorted by `ai_acceptance_score` (descending) to match the existing order.
- Follow the existing naming conventions for fields (snake_case).
